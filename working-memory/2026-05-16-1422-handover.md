## Architecture en place (pushed sur `claude/check-project-status-49qIQ`)

- 5 axes outlook dans `core-memory/outlook-axes.yaml`, lus par
  `tasks/outlook.py`. Cron daily 18h UTC ; à chaque run, regarde la date,
  match `day_of_month`, sinon skip silent.
- Pattern dual `=== TELEGRAM ===` / `=== ARCHIVE ===` GLOBAL dans le
  template `_PROMPT` (tasks/outlook.py). Le code Python split, envoie la
  partie TG sur Telegram, persiste la partie ARCHIVE dans
  `digests/outlook/<slug>.md` (append-only, header auto au premier run).
- Au run suivant : la queue de `digests/outlook/<slug>.md` est injectée
  dans le prompt sous "# Déjà proposé sur cet axe lors des runs précédents
  — NE PAS répéter". Mémoire long terme anti-redite.
- Filtre intelligent : `memory.tail_dated_entries()` skip les préambules /
  doc et ne retourne que les entrées datées (regex `## YYYY-MM-DD`).
  Utilisé pour findings.md, decisions.md, archives. findings.md vide =
  vraiment vide dans le prompt, pas la doc méta.

## Conventions établies pendant cette session

- **Marqueurs exacts** : `=== TELEGRAM ===` puis `=== ARCHIVE ===` sur leur
  propre ligne, pas de variation. Le parser Python est strict.
- **TG = condensé** pour la lecture sur téléphone. **ARCHIVE = exhaustif**,
  sert de source locale et de mémoire pour empêcher la redite.
- **Section "À planifier" finale** : par défaut le template global la
  demande (2-5 bullets actionnables). Override dans le focus de l'axe si
  l'axe a sa propre structure (cas de `serendipite` qui n'a pas de plan
  d'action).
- **Tier per-axis** dans le yaml (`tier: opus | sonnet`), surcharge le
  `model` de `schedule.yaml`.
- **`web_search_max` per-axis** dans le yaml, jamais hardcodé Python.
- **Préférences Sylvain confirmées** : contenu intellectuel EN strict,
  sorties physiques focus sud Seine-et-Marne avant Paris (sauf événement
  qui justifie vraiment le déplacement Paris). FR OK pour le contenu
  famille uniquement (axe `famille`).

Pour montrer le prompt rendu d'un axe (snippet réutilisable) :

```bash
uv run python <<'PY'
from datetime import datetime, timedelta, UTC
from tasks import outlook
from tasks._lib import memory, paths

SLUG = "famille"  # changer ici
axes_cfg = outlook._load_axes()
axis = next(a for a in axes_cfg["axes"] if a["slug"] == SLUG)
fake = datetime(2026, 6, axis["day_of_month"], 18, 0, tzinfo=UTC)
horizon = fake + timedelta(days=90)

print(outlook._PROMPT.format(
    title=axis["title"],
    slug=axis["slug"],
    date_today=fake.strftime("%d/%m/%Y"),
    date_horizon=horizon.strftime("%d/%m/%Y"),
    location=outlook._format_location(memory.read_location()),
    focus=axis["focus"].strip(),
    findings=memory.tail_dated_entries(paths.FINDINGS, max_chars=3000) or "(vide)",
    decisions=memory.tail_dated_entries(paths.DECISIONS, max_chars=2000) or "(vide)",
    archive_tail=memory.tail_dated_entries(outlook._archive_path(axis["slug"]), max_chars=6000) or "(rien — premier run sur cet axe)",
    ws=axis["web_search_max"],
    queries_hints=outlook._format_hints(axis.get("queries_hints")),
))
PY
```
