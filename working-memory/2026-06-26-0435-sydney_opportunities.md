**Crée le GitHub + un billet avant le 8 août.** Pas un grand papier : un repo avec quelques evals rudimentaires (tests de scheming/sandbagging sur deux ou trois modèles avec tes propres métriques poker — VPIP, agressivité intentionnelle vs accidentelle, bluff détecté), et 1000 mots sur LessWrong ou ton Substack qui posent le cadre (pro GTO → fraude OKEx → deception LLM → pourquoi les benchmarks actuels ratent le bluff intentionnel).
